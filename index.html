資料結構筆記 - 布隆過濾器 Bloom Filter

發表於 2024-01-10 分類於 DSA 文章字數： 651 所需閱讀時間 ≈ 2 分鐘

Summary

  • 用途: 用來快速判斷元素是否存在於一個集合中
  • 應用:
    • Check Duplicate: name、email 等是否已被使用
    • Filters: 過濾惡意請求 (ex: Cache penetration)、垃圾郵件等
  • 優點: 快速、節省空間
  • 缺點: 可能有 false positive (不存在但判定為存在)

原理

使用多個 hash function，將其映射到固定長度的 array 中，如果全部都命中表示可能存在。

  1. 由預期的 False positive rate (p) 以及存入的資料筆數 (n) 來求出 Array Size (m) 以及 Hash Functions數 (k)

    • False positive rate
    • Array size
    • Number of hash functions

  2. 選擇 Hash function
    可以使用 **Non-Cryptographic Hash (非密碼雜湊函數)**，雖然有 collision 的可能，但足夠好了，而且比較快。
    例如: MurmurHash、FarmHash、SpookyHash

前言

Flask和Django是Python中最多人使用的網頁框架(web framework)，雖然這些框架有自己的WSGI server，但僅適合用於開發測試階段，因此正式運行時通常會使用另外的WSGI server，如Gunicorn或uWSGI，除此之外，還會用Nginx來做反向代理，可以說一個最基本的Python網頁服務基本上就會是 web framework (Flask/Django) + WSGI server(Gunicorn/uWSGI) + Nginx。在這個架構之上，我們可以加上supervisor來管理伺服器進程，以防止網頁突然掛掉，最後，我們也可以將服務運行在docker container中，方便CI/CD，今天將介紹 Flask + gunicorn + supervisor + nginx + docker，共5個componet的網頁服務在Linux環境中如何建置。

Gunicorn

如同一開始所說的，Flask的WSGI不夠穩定，用於開發測試階段OK，但要用於生產環境還是不太行，這時候就要召喚Gunicorn來代替Flask運行我們的服務了。之所以選擇用Gunicorn其實沒有什麼特別理由，單純因為使用上比較簡單，不做額外設定的情況下只需要一行command line就能搞定。

  3. 設置參數

  • -b BIND, --bind=BIND: 綁定的Network socket，預設為127.0.0.1:8000

worker-class中，除了sync是同步方法，其餘都是屬於非同步方法，主要是實作方法的差異，使用時要安裝相應的套件，有興趣的可以參考這篇文章: gunicorn workers 区别(sync/eventlet/gevent/thread/tornado)

發表於 2023-03-29 分類於 Social Network Analysis 文章字數： 2.3k 所需閱讀時間 ≈ 8 分鐘

介紹

介數中心性（Betweenness Centrality）是社會網絡分析中常用的一種指標，用於度量一個節點在整個網絡中扮演了多重要的角色，簡單來說，介數中心性指標評估了一個節點在網絡中擔任了多少條最短路徑的中介者角色。

更白話一點，介數中心性看的是一個節點有多常被其他人經過，舉例來說：某些公車站點連接了不同的公車路線，這些公車站點對於連接整個公車網絡起著重要的作用，如果這些公車站發生了任何問題，整個公車路線都會受到影響。這些公車站點就是具有高介數中心性的節點。
