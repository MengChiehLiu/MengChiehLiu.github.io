flask-gunicorn-supervisor-docker

發表於 2023-05-15 分類於 Web Design 閱讀次數： 文章字數： 916 所需閱讀時間 ≈ 3 分鐘

前言

Flask和Django是Python中最多人使用的網頁框架(web framework)，雖然這些框架有自己的WSGI server，但僅適合用於開發測試階段，因此正式運行時通常會使用另外的WSGI server，如Gunicorn或uWSGI，除此之外，還會用Nginx來做反向代理，可以說一個最基本的Python網頁服務基本上就會是 web framework (Flask/Django) + WSGI server(Gunicorn/uWSGI) + Nginx。在這個架構之上，我們可以加上supervisor來管理伺服器進程，以防止網頁突然掛掉，最後，我們也可以將服務運行在docker container中，方便CI/CD，今天將介紹 Flask + gunicorn + supervisor + nginx + docker，共5個componet的網頁服務在Linux環境中如何建置。

Gunicorn

如同一開始所說的，Flask的WSGI不夠穩定，用於開發測試階段OK，但要用於生產環境還是不太行，這時候就要召喚Gunicorn來代替Flask運行我們的服務了。之所以選擇用Gunicorn其實沒有什麼特別理由，單純因為使用上比較簡單，不做額外設定的情況下只需要一行command line就能搞定。

    第一個app是py檔名，第2個app是Flask app的名字

  3. 設置參數

  • -b BIND, --bind=BIND: 綁定的Network socket，預設為127.0.0.1:8000
  • -w WORKERS, --workers=WORKERS: 進程數量，預設為1
  • -k STRING or --worker-class STRING: 運行模式，預設為sync，其他還有eventlet、gevent、tornado、gthread

worker-class中，除了sync是同步方法，其餘都是屬於非同步方法，主要是實作方法的差異，使用時要安裝相應的套件，有興趣的可以參考這篇文章: gunicorn workers 区别(sync/eventlet/gevent/thread/tornado)

  1. 使用config檔 -c CONFIG, --config=CONFIG: config檔路徑，預設為./gunicorn.conf.py

Supervisor

雖然我們可以在gunicorn中運行多個process或多個thread，但如果gunicorn掛掉那全部都掛了，為了防止這種情況，我們可以使用Supervisor來管理gunicorn(及其他服務)的運行，一旦gunicorn掛掉可以透過supervisor立即自動重啟。此外，Supervisor在Docker中也扮演了很重要的角色，如果你的網頁有一些獨立於網頁伺服器的服務，但又不想將這些服務分散在多個docker container的話，也可以透過Supervisor來同時管理的個資源，並運行在單一個docker container中。
